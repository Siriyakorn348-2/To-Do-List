# To-Do List

แอปพลิเคชัน To-Do List พัฒนาด้วย React + Vite

## ฟีเจอร์หลัก

- เพิ่มรายการงาน (Create Task)
- แสดงรายการงาน (List Tasks)
- แก้ไขรายการงาน (Edit Task)
- ลบรายการงาน (Delete Task)
- ทำเครื่องหมายว่างานเสร็จสิ้น/ยังไม่เสร็จ (Complete/Incomplete)
- จัดเก็บข้อมูลด้วย `localStorage`

## การติดตั้งโปรเจกต์

1. Clone โปรเจกต์จาก GitHub

```bash
git clone https://github.com/Siriyakorn348-2/To-Do-List.git
```

2. เข้าไปที่โฟลเดอร์โปรเจกต์

```bash
cd To-Do-List
```

3. ติดตั้ง dependencies

```bash
npm install
```

## การรันโปรเจกต์

```bash
npm run dev
```

## แนวคิดการออกแบบระบบโดยสรุป

โปรเจกต์นี้ออกแบบแบบ Component-Based โดยแยกหน้าที่ของแต่ละส่วนออกจากกันอย่างชัดเจน เพื่อให้โค้ดอ่านง่ายและดูแลต่อได้สะดวก

- `Todo.jsx` เป็นตัวกลางหลักของระบบ และเก็บ state ของรายการงานทั้งหมด
- `Form.jsx` ใช้สำหรับเพิ่ม task ใหม่
- `TodoList.jsx` ใช้แสดงรายการ task ทั้งหมด
- `TodoItem.jsx` ใช้แสดง task ทีละรายการ พร้อมปุ่มเช็คสถานะ แก้ไข และลบ
- `Footer.jsx` ใช้แสดงจำนวนงานที่ทำเสร็จแล้วและจำนวนงานทั้งหมด

การเก็บข้อมูลใช้ `localStorage` เพื่อให้ข้อมูลยังอยู่แม้รีเฟรชหน้าเว็บ โดยระบบจะทำงานดังนี้

- โหลดข้อมูลเดิมจาก `localStorage` ตอนเปิดแอป
- อัปเดตรายการใน React state เมื่อมีการเพิ่ม แก้ไข ลบ หรือเปลี่ยนสถานะงาน
- บันทึกข้อมูลล่าสุดกลับไปที่ `localStorage` อัตโนมัติเมื่อข้อมูลเปลี่ยน

โครงสร้างข้อมูลของ task แต่ละรายการเป็น object ที่มี

- `id` สำหรับระบุรายการแบบไม่ซ้ำกัน
- `name` สำหรับชื่อ task
- `done` สำหรับสถานะว่าเสร็จแล้วหรือยัง
